Mapcolonies : Usage Tips

Created by Shlomi Koncha, last modified by Alex Lebinson on Feb 24, 2022

Use WMTS layer extent

It is important to specify the extent as accurately as possible to the footprint of the desired layer in order to properly consume the existing tiles for the layer, it will improve response performance.

OpenLayers WMTS layer from Capabilities

There is an easy way to consume WMTS layer from GetCapabilities response with OpenLayers, and even create the configuration manually. see source - https://openlayers.org/en/latest/examples/wmts-layer-from-capabilities.html .

Tiling scheme - Cesium

Pay attention to define 2x1 tiling scheme (WGS84) and not 1x1 at client side (Cesium) in order to consume the layers properly

Query catalog by Product Type (mc:productType) - see valid values Map Colonies - Catalog Information

Users should usually search for “OrthophotoBest“ or “RasterVectorBest“ layers, other layers by specific need “Orthophoto“, “OrthophotoBest“ and “RasterVectorBest“ layers can be updated “behind“, invalidate and hard refresh browser cache if layer looks different from other users.
